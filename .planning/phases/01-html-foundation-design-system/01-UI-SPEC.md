## Spacing Scale

All values are multiples of 4. Declared as CSS Custom Properties in `:root` of `css/main.css`.

| Token | CSS Variable | px Value | rem Value | Usage |
|-------|-------------|----------|-----------|-------|
| xs | `--space-xs` | 4px | 0.25rem | Icon-to-label gap, tight inline padding |
| sm | `--space-sm` | 8px | 0.5rem | Compact element spacing, badge padding |
| md | `--space-md` | 16px | 1rem | Default element spacing, form field gaps |
| lg | `--space-lg` | 24px | 1.5rem | Card padding, nav item padding |
| xl | `--space-xl` | 48px | 3rem | Layout column gaps |
| 2xl | `--space-2xl` | 64px | 4rem | Section vertical padding (desktop) |
| 3xl | `--space-3xl` | 64px | 4rem | Major section breaks (desktop hero) — same as 2xl; use contextually |

**Exceptions:**
- Touch targets (nav links, CTA buttons, hamburger toggle): minimum 44px height — enforced via `min-height: 44px` on interactive elements, not via spacing token. Required for WCAG 2.5.5 Target Size.
- Scroll padding: `scroll-padding-top: 80px` (accounts for sticky header height) — declared on `:root` or `html`.
- Form inputs: `font-size: 16px` minimum — this is a typography rule, not spacing, but declared unconditionally to prevent iOS Safari auto-zoom (FOUND-02, CALC-05, FORM-02).

**Source:** STACK.md §1 Custom Properties Pattern. Exceptions from SUMMARY.md §Critical Pitfalls #4, REQUIREMENTS.md CALC-05/FORM-02.

## Typography

| Role | CSS Variable | Size | Weight | Font | Line Height | Usage |
|------|-------------|------|--------|------|-------------|-------|
| Body | `--font-size-base` | 16px | 400 | Be Vietnam Pro | 1.6 | All paragraph text, form labels, nav links |
| Label / Small | `--font-size-sm` | 14px | 400 | Be Vietnam Pro | 1.4 | Disclaimers, captions, microcopy, badge text |
| Subheading | `--font-size-xl` | 20px | 600 | Be Vietnam Pro | 1.3 | Section subheadings (`<h3>`), card titles |
| Heading / Display | `--font-size-2xl` | `clamp(28px, 5vw, 40px)` | 600 / 800 | Be Vietnam Pro (`<h2>`) / Montserrat (`<h1>`) | 1.2 / 1.1 | `<h2>` section headings at weight 600 Be Vietnam Pro; `<h1>` hero display at weight 800 Montserrat |

**Weights declared:**
- Be Vietnam Pro: 400 (regular) and 600 (semibold) — maximum 2 weights for this family.
- Montserrat: 800 (extra-bold) — 1 weight for this family.

**Responsive display:** The `clamp(28px, 5vw, 40px)` value on `--font-size-2xl` means `<h1>` renders at 28px on a 375px viewport and scales up to 40px at 1200px+. `<h2>` section headings use the same token, which resolves to approximately 28–32px at typical viewport widths — acceptable for a section heading hierarchy. All other sizes are fixed.

**Primary focal point:** Hero `<h1>` with Montserrat 800 display weight is the primary focal anchor on every page load. The secondary anchor is `.btn--primary` gold CTA positioned directly below the `<h1>` in the hero section.

**Form input rule:** All `<input>`, `<select>`, `<textarea>` must use `font-size: 16px` unconditionally — no exceptions. Prevents iOS Safari viewport zoom on focus.

**Vietnamese diacritics:** Both fonts loaded with `subset=vietnamese` parameter. `display=swap` prevents invisible text flash.

**Source:** STACK.md §1 (token definitions), §3 (font selection). Display size exceptions from SUMMARY.md §Critical Pitfalls #5. iOS zoom rule from REQUIREMENTS.md FOUND-07, CALC-05. Typography fix from UI checker revision 2026-03-28.

### 60/30/10 Distribution

| Role | Value | Coverage |
|------|-------|---------|
| Dominant (60%) | `#FFFFFF` / `#F7F5F0` | Main content surfaces, all white-bg sections |
| Secondary (30%) | `#0D2B5E` / `#091D42` | Header, footer, hero bg, dark callout sections |
| Accent (10%) | `#C9A84C` | See reserved list below |

### Accent (`#C9A84C`) Reserved Exclusively For

1. Primary CTA button background (`bg #C9A84C`, text `#091D42`)
2. Product card left-border accent (4px solid gold border per REQUIREMENTS.md PROD-03)
3. Hero `<h1>` text when rendered on navy background (use `#E2C47A` gold-light for this case)
4. Phosphor icon fill color in services cards (on white bg, gold at ~2.9:1 — use `#A8863A` gold-dark instead to reach 4.5:1 on white)
5. Hover underline/indicator on nav links
6. Footer link color (on `#091D42` navy-dark background — contrast ~5.8:1, passes AA)

**CRITICAL constraint:** Gold (`#C9A84C`) MUST NOT appear as text on white (`#FFFFFF`) or off-white (`#F7F5F0`). Contrast ratio is ~2.9:1, which fails WCAG AA. On white surfaces, use `#A8863A` (gold-dark) for icon fills, or switch the icon/text to `#0D2B5E` navy. This constraint applies everywhere in every phase.

**Destructive:** `#C0392B` — reserved for inline form validation error messages only. No destructive actions exist in Phase 1 (scaffold only). Declared in `:root` for later phases.

**Source:** STACK.md §5 (palette with hex values, contrast table, application guide). Critical constraint from SUMMARY.md §Critical Pitfalls #6. REQUIREMENTS.md PROD-03.

## CDN Reference (Phase 1 Deliverables)

Phase 1 must embed these CDN links in `<head>` of `index.html`:

```html
<!-- Fonts: preconnect + Google Fonts with Vietnamese subset -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Be+Vietnam+Pro:wght@400;600&family=Montserrat:wght@800&display=swap&subset=vietnamese" rel="stylesheet">

<!-- Phosphor Icons v2 — CSS bold variant (no JS required) -->
<link rel="stylesheet" href="https://unpkg.com/@phosphor-icons/web@2.1.1/src/bold/style.css">
```

**Version pinning:** Phosphor Icons is pinned to `@2.1.1`. Do not use `@latest` in production — unpkg `@latest` resolves at request time and can introduce breaking changes.

**Font weight reduction:** Be Vietnam Pro loads only weights 400 and 600 (dropped 500 and 700). Montserrat loads only weight 800 (dropped 700). This reduces font payload and enforces the 2-weight-per-family contract.

**Source:** STACK.md §3 embed code, §4 Phosphor CDN. Font weights updated per UI checker revision 2026-03-28. Note from STACK.md: CDN URLs should be verified against current Phosphor Icons releases before use.
